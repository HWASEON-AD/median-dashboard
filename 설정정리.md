### naver_check.py 핵심 로직 (2026-06-23 디버깅으로 확정)
- 네이버 **모바일 통합검색**에서 발행URL이 검색결과(인기글 등)에 있으면 '노출중', 없으면 '미노출'
- **인기글(UGC) 블록은 client-side JS로 늦게 렌더됨** → `_wait_for_ugc()`로 카페/블로그 링크 3개 이상 렌더될 때까지 대기 후 매칭 (이거 빠지면 노출글을 미노출로 오탐 — 과거 버그)
- 캡처: 매칭된 글 카드를 **화면 중앙(block:'center')으로 스크롤** → 빨간 테두리 → **전체화면(뷰포트) 캡처**. 카드가 가운데 와서 안 잘리고 주변 검색결과 맥락까지 보임

아모스에서 개발 후 이식. 커밋 `a50364f`(합산·캐시) → `d0a865b`(총조회수 단일화) → `6e38d34`(지식인) → `8f985c7`(캡처추가).

### 신규 컬럼 (median_posts, 사용자가 SQL Editor에서 직접 실행)
- `past_urls` / `past_image_host_urls` / `past_hwaseon_urls` — 콤마 구분 text
- `search_volume_pc` / `search_volume_mobile` / `search_volume_at` — 검색량 일일 캐시

### 총 조회수 (컬럼 1개)
```
총 조회수 = 카페 readCount 합(현재+과거 URL)
          + 지식인 조회수 합(현재+과거 URL)
          + 이미지호스팅 조회수 합(현재+과거)
          + views_base (라이브로 못 읽는 과거분)
```
- 카페+지식인 합계는 `cafe_views`에, 이미지호스팅 합계는 `image_views`에 `/api/refresh-views`가 저장.
- 예전의 `조회수`(combined) / `총 조회수`(image_views 원본) **2개 컬럼은 1개로 통합**. `image_views` 원본 표시는 삭제.
- `computeSnapshot()` / `views_offset` **폐기** — PATCH가 `views_base`를 덮어쓰던 위험 제거.
- 클릭수 = `hwaseon_url` + `past_hwaseon_urls`의 단축코드 전부 합산 (`/api/clicks?code=a,b,c`).

### 노출 확인 / 캡처 (scripts/naver_check.py)
- 발행URL → `past_urls` 순차 매칭, **첫 매칭에서 중단** + 그 글 캡처 1장. 검색은 키워드당 1회.
- `parse_url()`에 **지식인(kin.naver.com docId)** 파싱 추가 (기존엔 post_no 못 뽑아 영구 미노출).

### 검색량 일일 캐시
- 기준일 = **KST 06:00** 넘어갈 때 갱신(6시 이전이면 어제 날짜). 같은 기준일이면 DB 캐시 즉시 반환.
- 신규 키워드만 그때 1회 조회 후 기록. 캐시 기록은 병렬(`Promise.all`).

### 캡처 수동 추가 (`POST /api/captures/add`)
- 캡처 탭 `+ 캡처 추가` → 날짜 / 브랜드 / 제품 / 검색어 / 노출탭(자동, 읽기전용) / 기본·전체보기 / 이미지 파일.
- 저장 시 `median_daily_captures` 생성 + `median_daily_exposure` upsert(중복 무시) → 총 노출일 연계.
- 같은 (post_id, date) 캡처가 있으면 새 행 없이 이미지만 갱신(멱등).
- 키워드는 `median_posts`에서만 선택 (임의 post_id 주입 방지).

### 🚨 이번에 발견한 중대 버그 2개 (아모스와 동일하게 수정)
1. **Next.js Data Cache가 supabase-js의 GET을 캐싱** → DB를 갱신해도 select가 옛 값 반환. `.next/cache`에 남아 서버 재시작에도 생존.
   해결: `lib/supabase.ts`에서 모든 Supabase 요청에 `cache: 'no-store'` 강제.
2. **PostgREST 기본 1000행 제한** → 노출기록이 1000건을 넘으면 잘려서 `총 노출일`이 틀리게 나옴.
   해결: `lib/fetch-all.ts`의 `fetchAllRows()` range 페이지네이션. `keywords GET`, `export-excel` 적용.

### ⚠️ 운영 주의
- 관리자 화면에서 발행URL을 바꿔도 **옛 URL이 자동으로 `past_urls`로 이관되지 않는다** (엑셀 임포트만 자동). 직접 지난URL 칸에 넣어야 조회수 합산이 유지된다.
- 삭제된 카페글의 마지막 값을 `views_base`로 옮길 땐 반드시 `cafe_views`를 null로 비울 것 (안 그러면 `refresh-views`가 옛 값을 유지해 이중계상).
- **지식인 조회수는 조회할 때마다 그 글 조회수가 1 증가**한다 (HTML 페이지를 실제로 열어야 해서). 크론 3회/일 → 하루 +3 자체 증가.
- `median_posts` 행 삭제 시 `median_daily_exposure`가 FK cascade로 함께 삭제된다. 운영 중 DELETE 금지.
- **헤더 클릭 정렬은 아모스 전용** — 메디안엔 없음. 필요 시 이식.
