## 캡처 자동화 (아모스와 완전 분리)
- 전용 테이블: median_posts / median_daily_exposure / median_daily_captures
- 전용 스토리지 버킷: median-captures
- 체크 스크립트: scripts/naver_check.py (median_* 전용, `--post-id <uuid>` 단일모드 지원)
- 워크플로: .github/workflows/naver_daily_check.yml (workflow_dispatch + post_id 입력)
- 스케줄: cron-job.org가 정확한 시각에 workflow_dispatch 호출 (사용자 등록 필요)

- **분리 불변식**: 메디안 코드/스크립트/워크플로는 amos_* 테이블·amos-captures 버킷·amos 워크플로를 절대 참조하지 않는다.

## 최초 1회 셋업 (코드 외 작업)
1. Supabase SQL Editor에서 scripts/median_daily_captures.sql 실행 (캡처 테이블 생성)
2. GitHub repo에 Secret SUPABASE_SERVICE_KEY 등록
3. Vercel에 GH_DISPATCH_TOKEN 환경변수 추가
4. cron-job.org에 메디안 워크플로 dispatch 스케줄 등록
